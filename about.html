Philosophy

How I approach software.

Six principles that shape every build. No marketing fluff — just how I actually work.

01Native first. If it runs on the platform, it should feel like it belongs there.
02Simple software. One job, done well. No feature creep, no bloat.
03Automation over repetition. If I do it twice, the third time is a script.
04Maintainable systems. Code I can still understand six months later.
05Finish before expanding. Ship v1, then decide if v2 is worth it.
06Build tools I actually use. If it doesn't earn its place, it doesn't ship.
